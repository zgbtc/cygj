# Crypto Tools Hub 使用说明

## 项目概述

Crypto Tools Hub 是一个加密工具聚合平台，目前已集成 **Stealth Transfer** 混币器工具。

## 当前状态

✅ **已完成**:
- Next.js 网站框架搭建
- 首页工具展示
- Stealth Transfer 详情页
- 响应式设计
- 工具分类和搜索功能

🚀 **已启动的服务**:
- Next.js 开发服务器: http://localhost:3000
- Flask API 服务器: http://localhost:5000

## 访问方式

### 1. 访问主网站
打开浏览器访问: **http://localhost:3000**

你会看到:
- 精美的首页设计
- 6 个工具卡片（1 个可用，5 个即将推出）
- 工具分类（隐私工具、DeFi 工具、数据分析、钱包工具）
- 搜索功能

### 2. 访问 Stealth Transfer 工具

有两种方式:

**方式 1: 通过网站导航**
1. 访问 http://localhost:3000
2. 点击 "Stealth Transfer" 卡片上的 "立即使用" 按钮
3. 进入工具详情页
4. 点击 "🚀 立即使用" 按钮
5. 跳转到 Flask 提供的工具界面 (http://localhost:5000/stealth.html)

**方式 2: 直接访问**
- 详情页: http://localhost:3000/stealth-transfer
- 工具界面: http://localhost:5000/stealth.html

## 功能说明

### 首页功能

1. **工具展示**
   - 显示所有可用和即将推出的工具
   - 每个工具卡片包含：图标、名称、描述、支持的链、核心特性、用户评分

2. **分类筛选**
   - 点击分类卡片可以筛选对应类别的工具
   - 支持的分类：隐私工具、DeFi 工具、数据分析、钱包工具

3. **搜索功能**
   - 在搜索框输入关键词
   - 实时搜索工具名称、描述和特性

### Stealth Transfer 详情页

1. **工具介绍**
   - 用户评分、活跃用户数、跳数范围
   - 工作原理说明（3 步流程）
   - 核心特性展示

2. **费用说明**
   - 详细的费用表格
   - 不同跳数对应的服务费和 Gas 费
   - 推荐使用 100 跳（性价比最高）

3. **快速访问**
   - "立即使用" 按钮 → 跳转到工具界面
   - "查看文档" 按钮 → 跳转到 GitHub 文档

### Stealth Transfer 工具界面

访问 http://localhost:5000/stealth.html 使用混币器:

1. **连接钱包** 或 **输入私钥**
2. **选择链**: BSC Testnet / BSC Mainnet / Ethereum
3. **输入参数**:
   - 总金额
   - 目标地址
   - 跳数（10-1000）
   - 助记词（可选，自动生成）
4. **查看费用估算**
5. **执行混币**

## 测试建议

### 测试网测试

建议先在 BSC Testnet 测试:

1. **获取测试币**
   - 访问 BSC Testnet Faucet
   - 获取一些 tBNB

2. **测试参数**
   - 金额: 0.1 tBNB
   - 跳数: 10 或 100
   - 使用测试助记词: `decline cluster album give brief scrap apart onion dust donor figure primary`

3. **观察结果**
   - 查看交易哈希
   - 在 BSCScan Testnet 查看交易记录
   - 确认目标地址收到资金

## 开发说明

### 启动服务

```bash
# 终端 1: 启动 Next.js
cd crypto-tools-hub
npm run dev

# 终端 2: 启动 Flask API
cd stealth_transfer
python app.py
```

### 停止服务

在对应的终端按 `Ctrl+C`

### 修改代码

- **前端修改**: 编辑 `crypto-tools-hub/app/` 下的文件，保存后自动热重载
- **后端修改**: 编辑 `stealth_transfer/` 下的文件，保存后 Flask 自动重启

## 常见问题

### Q: 为什么点击"立即使用"没反应？
A: 确保 Flask 服务器正在运行 (http://localhost:5000)

### Q: 如何添加新工具？
A: 
1. 在 `app/page.tsx` 的 `tools` 数组中添加工具信息
2. 创建工具详情页 `app/[tool-name]/page.tsx`
3. 如果需要后端，在 Flask 中添加 API 端点

### Q: 如何修改样式？
A: 使用 Tailwind CSS 类名，或在 `app/globals.css` 中添加自定义样式

### Q: 如何部署？
A: 
- 前端: `npm run build` 然后部署到 Vercel/Netlify
- 后端: 使用 Gunicorn + Nginx 部署 Flask 应用
