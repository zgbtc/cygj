## 前端部署 (Vercel)

### 方式 1: 通过 Vercel Dashboard

1. **推送代码到 GitHub**
   ```bash
   cd crypto-tools-hub
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/your-username/crypto-tools-hub.git
   git push -u origin main
   ```

2. **在 Vercel 导入项目**
   - 访问 https://vercel.com
   - 点击 "New Project"
   - 导入你的 GitHub 仓库
   - 选择 `crypto-tools-hub` 目录作为根目录
   - 点击 "Deploy"

3. **配置环境变量**
   在 Vercel 项目设置中添加:
   ```
   NEXT_PUBLIC_API_URL=https://your-api-domain.com
   ```

### 方式 2: 通过 Vercel CLI

```bash
# 安装 Vercel CLI
npm i -g vercel

# 登录
vercel login

# 部署
cd crypto-tools-hub
vercel

# 生产部署
vercel --prod
```

## 后端部署 (Flask API)

### 选项 1: 使用 Railway

1. **准备项目**
   ```bash
   cd stealth_transfer
   
   # 创建 requirements.txt
   pip freeze > requirements.txt
   
   # 创建 Procfile
   echo "web: gunicorn app:app" > Procfile
   
   # 安装 gunicorn
   pip install gunicorn
   ```

2. **部署到 Railway**
   - 访问 https://railway.app
   - 连接 GitHub 仓库
   - 选择 `stealth_transfer` 目录
   - 自动部署

3. **配置环境变量**
   在 Railway 项目设置中添加:
   ```
   FLASK_ENV=production
   BSC_RPC_URL=https://bsc-dataseed.binance.org/
   ETH_RPC_URL=https://eth-mainnet.g.alchemy.com/v2/YOUR_KEY
   ```

### 选项 2: 使用 DigitalOcean/AWS/阿里云

1. **创建服务器**
   - Ubuntu 22.04 LTS
   - 至少 1GB RAM
   - 开放端口 80, 443

2. **安装依赖**
   ```bash
   # 更新系统
   sudo apt update && sudo apt upgrade -y
   
   # 安装 Python 和 pip
   sudo apt install python3 python3-pip python3-venv -y
   
   # 安装 Nginx
   sudo apt install nginx -y
   
   # 安装 Supervisor
   sudo apt install supervisor -y
   ```

3. **部署应用**
   ```bash
   # 克隆代码
   cd /var/www
   git clone https://github.com/your-username/crypto-tools-hub.git
   cd crypto-tools-hub/stealth_transfer
   
   # 创建虚拟环境
   python3 -m venv venv
   source venv/bin/activate
   
   # 安装依赖
   pip install -r requirements.txt
   pip install gunicorn
   ```

4. **配置 Gunicorn**
   创建 `/etc/supervisor/conf.d/stealth-api.conf`:
   ```ini
   [program:stealth-api]
   directory=/var/www/crypto-tools-hub/stealth_transfer
   command=/var/www/crypto-tools-hub/stealth_transfer/venv/bin/gunicorn -w 4 -b 127.0.0.1:5000 app:app
   user=www-data
   autostart=true
   autorestart=true
   stderr_logfile=/var/log/stealth-api.err.log
   stdout_logfile=/var/log/stealth-api.out.log
   ```

5. **配置 Nginx**
   创建 `/etc/nginx/sites-available/stealth-api`:
   ```nginx
   server {
       listen 80;
       server_name api.your-domain.com;
   
       location / {
           proxy_pass http://127.0.0.1:5000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```

6. **启用配置**
   ```bash
   # 启用 Nginx 配置
   sudo ln -s /etc/nginx/sites-available/stealth-api /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   
   # 启动 Supervisor
   sudo supervisorctl reread
   sudo supervisorctl update
   sudo supervisorctl start stealth-api
   ```

7. **配置 SSL (Let's Encrypt)**
   ```bash
   # 安装 Certbot
   sudo apt install certbot python3-certbot-nginx -y
   
   # 获取证书
   sudo certbot --nginx -d api.your-domain.com
   
   # 自动续期
   sudo certbot renew --dry-run
   ```

### 选项 3: 使用 Docker

1. **创建 Dockerfile**
   在 `stealth_transfer/` 目录创建:
   ```dockerfile
   FROM python:3.10-slim
   
   WORKDIR /app
   
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   RUN pip install gunicorn
   
   COPY . .
   
   EXPOSE 5000
   
   CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:5000", "app:app"]
   ```

2. **创建 docker-compose.yml**
   ```yaml
   version: '3.8'
   
   services:
     api:
       build: ./stealth_transfer
       ports:
         - "5000:5000"
       environment:
         - FLASK_ENV=production
       restart: always
   ```

3. **部署**
   ```bash
   docker-compose up -d
   ```

## 环境变量配置

### 前端 (.env.local)
```env

# 分析工具 (可选)
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
```

### 后端 (.env)
```env
# Flask 配置
FLASK_ENV=production
SECRET_KEY=your-secret-key-here

# RPC 节点
BSC_MAINNET_RPC=https://bsc-dataseed.binance.org/
BSC_TESTNET_RPC=https://data-seed-prebsc-1-s1.binance.org:8545/
ETH_MAINNET_RPC=https://eth-mainnet.g.alchemy.com/v2/YOUR_KEY

FEE_ADDRESS=0xe602348170bc045c588bf1638b0edc592f767250

# CORS 配置
CORS_ORIGINS=https://your-domain.com,https://www.your-domain.com
```

## 性能优化

### 前端优化
1. **启用 CDN**
   - Vercel 自动提供全球 CDN
   
2. **图片优化**
   - 使用 Next.js Image 组件
   - 启用 WebP 格式

3. **代码分割**
   - Next.js 自动代码分割
   - 动态导入大型组件

### 后端优化
1. **使用 Redis 缓存**
   ```python
   from flask_caching import Cache
   
   cache = Cache(app, config={'CACHE_TYPE': 'redis'})
   ```

2. **增加 Worker 数量**
   ```bash
   gunicorn -w 8 -b 0.0.0.0:5000 app:app
   ```

3. **使用负载均衡**
   - Nginx 负载均衡
   - 多个 Gunicorn 实例

## 安全建议

1. **API 安全**
   - 启用 CORS 白名单
   - 添加 Rate Limiting
   - 使用 API Key 认证

2. **服务器安全**
   - 配置防火墙 (UFW)
   - 定期更新系统
   - 使用 Fail2ban 防止暴力攻击

3. **密钥管理**
   - 不要在代码中硬编码密钥
   - 使用环境变量
   - 定期轮换密钥
